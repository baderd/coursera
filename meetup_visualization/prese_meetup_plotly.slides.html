What is Plotly?¶

  • JavaScript graphing library plotly.js is build on top of D3 --> interactivity!
  • plotly.js went open-source in Nov 2015
  • graphs are specified declaratively in JSON format
In [ ]:
{
    "data": [
        {
            "x":[1,2,3],
            "y":[2,3,4]
        }
    ],
    "layout" : {
        "title":"Plotly Plot"
    }
}

http://www.plot.ly

What else is Plotly?¶

  • API libraries for Python, R, Julia, Scala, ...
  • directly convert ggplot, pandas or matplotlib plots into interactive D3 plots
  • embed interactive Plotly plots into
    • jupyter/ipython notebooks
    • knitr reports
    • Shiny Apps

https://plot.ly/api/

Remember ggplot from last meetup? Now comes: ggplotly¶

ggplotly works by serializing ggplot2 figures into plotly's universal graph JSON.

Create a sophisticated plot in R with ggplot...

In [4]:
%%R
library(ggplot2)

d <- diamonds[sample(nrow(diamonds), 1000), ]
p <- ggplot(data = d, aes(x = carat, y = price)) +
        geom_point(size = 1) +
        geom_smooth(aes(colour = cut, fill = cut)) + 
        facet_wrap(~ cut)
print(p)

And convert this plot into a fancy interactive plot with just one command...

In [5]:
%%R 
library(plotly)

ply = ggplotly(p)
saveWidget(ply, "py01_diamonds.html")
In [6]:
show_plotly_html("py01_diamonds.html")

OUTLINE: Grammar of Graphics¶

The Grammar of Graphics is a visualization theory developed by Leland Wilkinson in 1999.

  • influenced the development of graphics and visualization libraries alike
  • focus on 3 of its key principles
    • Separation of data from aesthetics (e.g. in ggplot)
    • Definition of common plot/chart elements
    • Composition of these common elements

Building blocks of visualizations¶

  • visual objects used to encode quantitative values
    • dots
    • lines
    • shapes
  • data types
    • quantitative
    • categorical

Hello world example in R¶

In [7]:
%%R
df <- data.frame(
  catdata= c("A", "B", "C"),
  quantdata = c(1,2,3)
)
p <- subplot(
  plot_ly(df, x=~catdata, y=~quantdata, type='scatter', mode='markers'), 
  plot_ly(df, x=~catdata, y=~quantdata, type='scatter', mode='markers+lines'),
  plot_ly(df, x=~catdata, y=~quantdata, type='bar')
)
saveWidget(p, "pr01_triple_basics.html")

p <- plot_ly(
    subset(poke_stats, Name %in% c('Squirtle','Bulbasaur','Charmander')), 
     x = ~Name, 
     y = ~Attack, type = 'bar', name="Att"
) %>% add_trace(
    y=~Defense, name='Def'
) %>% layout(
    barmode = 'stack'
)
saveWidget(p, "pr_poke_stackedbar.html")
